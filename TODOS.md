## 🔵 P2 - Medium（优化项，MVP 后处理）

### [T6] Week 3+: OCR 缓存优化

**状态：** 🔵 延后

**优先级：** P2 - Medium

**工作量：** 4-6 小时

**依赖：** 无

**背景：**
OCR API 按调用次数收费。如果用户重复上传相同图片，会重复收费。需要添加缓存机制。

**具体任务：**
- [ ] 引入 Redis（或使用小程序本地缓存）
- [ ] 实现图片 hash 缓存（MD5 或 SHA256）
- [ ] 添加缓存失效逻辑（24 小时过期）
- [ ] 成本监控和统计

**成功标准：**
- ✅ 相同图片的 OCR 结果被缓存
- ✅ 缓存命中率 >30%
- ✅ OCR 成本降低
