## 风险管理

### 技术风险
| 风险 | 缓解措施 |
|------|---------|
| EduKG API限制 | 本地缓存、请求合并、提前申请合作账号 |
| 音色克隆慢 | 异步处理+通知、提供音色样本库 |
| 小程序性能 | 分包加载、懒优化、本地缓存策略 |

### 进度风险
| 风险 | 缓解措施 |
|------|---------|
| 需求变更 | 锁定MVP范围，其他需求放入后续阶段 |
| 人员变动 | 文档完善、代码review、知识共享 |
| 测试时间不足 | 每个功能开发时同步测试，不累积到最后 |
